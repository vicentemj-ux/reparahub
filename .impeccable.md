## Design Context

### Users
- **Who:** Small-to-medium repair shop owners (cell phones, computers, consoles, electronics) and their technicians/staff in LATAM (primary: Mexico, early validation: Uruguay).
- **Context:** Using the dashboard daily to manage repairs, inventory, sales (POS), and customer relationships. Often on mobile devices or desktop in a workshop environment. Need to print thermal labels and receipts.
- **Job to be done:** Quickly find information, process repairs, manage stock, close daily cash, and communicate with customers. Speed and clarity are paramount.
- **Emotions:** Confidence that the system is reliable and professional. Pride in using a tool that looks polished and modern for their business.

### Brand Personality
- **Voice:** Professional, Confiable, Directo.
- **3-word personality:** Profesional / Confiable / Premium.
- **Emotional goals:** When a shop owner opens TallerCloud, they should feel "this is a serious, professional tool that I can trust with my business." Not intimidating, not playful — grounded and capable.

### Aesthetic Direction
- **Visual tone:** Premium / Glass-inspired. Subtle gradients, glassmorphism effects, micro-animations on interactions. NOT flat or sterile.
- **References:** Think Vercel Dashboard meets Linear — dark accents with glass panels, subtle depth through shadows and blur, but with enough visual weight and contrast to feel substantial.
- **Anti-references:** Apple/Stripe/Linear minimalism (too ethereal, too much whitespace, soft fonts, everything floating). Also NOT: phpMyAdmin/old ERP ugly tables, neon colors, aggressive dark mode.
- **Theme:** Light mode primary, with glass/depth effects on cards and modals. Dark mode exists but is secondary.

### Design Principles

1. **Professional Weight** — Elements should feel solid and grounded. Cards have visible borders AND shadows. Typography uses semibold/black weights for hierarchy. Nothing should feel floaty or ethereal.

2. **Glass Depth** — Cards, modals, and overlays use subtle backdrop-blur and transparency to create depth. Background gradients on headers and stat tiles add premium feel without being flashy.

3. **Functional Clarity First** — Every pixel serves a purpose. Information hierarchy is clear: bold/large for primary data, regular/small for secondary. Thermal print output must remain crisp and legible.

4. **Mobile-Native Responsive** — Touch targets >=44px. Tables scroll horizontally. Sidebar collapses to sheet. Forms stack vertically. The experience must feel native on both a phone and a desktop.

5. **Consistent Accent** — `blue-600` is the single accent color for all interactive elements (buttons, links, focus rings). Semantic colors (emerald, amber, red) only for status indicators. Never decorative.

### Current Tech Stack
- Next.js 16 (App Router) + React 19 + TypeScript
- Tailwind CSS 4 + shadcn/ui (New York style, neutral base)
- Framer Motion for animations
- Sonner for toasts (custom dark pill style)
- Lucide for icons
- Inter font family (400, 600, 700, 900 weights)
- OKLCH color tokens with blue-600 primary accent

### Spacing & Layout
- Page container: `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 py-6`
- Cards: `rounded-2xl border shadow-sm` with `p-6` internal padding
- ModuleHeader: `rounded-2xl border bg-white p-4 sm:p-8` with icon + title (extrabold italic) + eyebrow (10px uppercase tracking-widest)
- Stat tiles: `rounded-2xl` with colored backgrounds (`bg-blue-50`, `bg-emerald-50`, etc.)
- Border radius tokens: 6px (sm), 8px (md), 10px (lg), 14px (xl)

### Typography
- Primary: Inter (regular 400, semibold 600, bold 700, black 900)
- Monospace: Geist Mono
- Module titles: `text-xl font-extrabold italic tracking-tight`
- Eyebrows/labels: `text-[10px] uppercase tracking-widest`
- Stat numbers: `text-lg font-black tabular-nums`
- Body: `text-sm leading-relaxed text-muted-foreground`
